## 1. 배경 및 과제 목적

### 문제
- 부서별 원가동인 판단이 **경험·엑셀·개인 지식**에 의존
- 동일 계정이라도 부서·맥락에 따라 동인이 달라 **일관성·재현성** 확보가 어려움
- AI 도입 시에도 **회계사 최종 판단**과 **감사 가능한 이력**이 필요

### PoC 목표
1. 부서 비용 파일 업로드 → **계정 인벤토리·공통/특정 분리** 자동화
2. 계정별 **비용 성격 설명** + **원가동인 1~3순위** 추천 (근거 포함)
3. **회계사 검토·승인·수정·직접입력** UI 제공
4. 확정 결과를 **마스터 JSON + 엑셀**로 산출

## 2. 전체 프로세스

| Phase | 내용 | 자동/AI | 회계사 |
|-------|------|---------|--------|
| **0** | 계정 인벤토리 스캔 (`accounts_master.json`) | ✅ | — |
| **0.5** | 공통/특정 분리 (2개 이상 부서 등장 = 공통, 1개 = 특정) | ✅ | — |
| **1** | 4-type 분류 + 원가동인 1~3순위 추천 | AI (또는 PoC용 자동 추천) | 검토 |
| **확정** | 1순위 승인 / 다른 순위 선택 / 직접 입력 | — | ✅ |
| **출력** | `accounts_master.json` 갱신 + 부서별 최종 엑셀 | ✅ | 저장 버튼 |

**Phase 0.5 판단 기준 (현재 PoC):**
- **등장 부서 수**만 사용 (2개 이상 → 공통계정)
- 부서별 **금액 편차**는 조직 규모 차이로 볼 수 있어 공통/특정 판단에서 **제외**

## 3. 제공 도구 2종

### A. Streamlit — 회계사 확정 Workbench (실무 PoC)

**역할:** 실제 검토·확정·저장

**주요 기능:**
- 사이드바: 부서 CSV/XLSX **다중 업로드** → Phase 0 + 0.5 자동 실행
- 「전체」 탭: 공통계정 → 특수계정 → 추가판단 필요 계정 순 확정
- 모든 계정: **비용 성격 설명** (자동 규칙 / AI 분류근거)
- 명확 계정: **원가동인 1~3순위** (AI 추천 또는 PoC 자동 추천)
- **1순위 승인**, popover 2·3순위, **직접 입력**
- **「전체 저장」** → `accounts_master.json` + 부서별 `batch_*_final.xlsx`

**로컬 실행 (제출자 PC):**
```bash
cd cost-driver-agent
pip install -r requirements.txt
streamlit run streamlit_app/app.py --server.port 8642
```

**검증용 자료 (제출자가 Notion에 첨부):**

- [ ] 스크린샷 5~8장 (아래 체크리스트 참고)

### B. Vite React — 스토리텔링 데모 (프로세스 설명용)

**역할:** “왜 AI만으로 끝나면 안 되는지”, 회사 협의·배분 왜곡까지 **End-to-End 스토리** 시연

**6단계 Wizard:**
1. **업로드** — 비용 파일 + 컬럼 매핑
2. **AI 분석** — 비용 성격·동인 후보 (로컬 규칙 + AI 병합)
3. **회계사 1차 검토** — 동인 확정, 기타(직접 입력)
4. **회사(현업) 협의** — 부서 피드백·동인 조정·보류 처리
5. **부서 가중치** — 배분 기준 입력
6. **배분 실행** — 왜곡 분석 차트 + Excel export

**로컬 실행:**
```bash
cd cost-allocation-project
npm install
npm run dev   # http://localhost:5173
```

**검증용 자료:**
- [ ] `demo_vite.mp4` — 3분 녹화 (선택)
- [ ] `회계사_1차검토요청.xlsx` / `최종_확정본.xlsx` export 샘플

## 7. 설계 원칙 (회계 실무 관점)

1. **AI ≠ 최종 결정** — 추천·근거만 제공, 확정은 회계사
2. **공통계정 1회 확정 → 전 부서 반영** — 중복 판단 방지
3. **추가판단 필요 계정 분리** — 4-type·동인을 사람이 직접 기입
4. **감사 가능성** — JSON 마스터 + 엑셀 + (확정 시) 처리경로 기록
5. **PoC 한계 명시** — ERP 미연동, 샘플 데이터, Phase 1 AI는 환경에 따라 자동 추천 대체 가능

## 8. PoC 범위 및 한계

| 포함 | 미포함 |
|------|--------|
| CSV/XLSX 업로드 | ERP·회계 시스템 실시간 연동 |
| 공통/특정 자동 분리 | 법적·감사 최종 판단 |
| 비용 성격 자동 설명 (20+ 규칙) | 전 계정 100% AI 분류 (일부 규칙 기반) |
| 동인 1~3순위 추천 | 동인 확정 without 회계사 |
| 회계사 확정 UI + 엑셀 출력 | SaaS 멀티테넌트·권한 관리 |
