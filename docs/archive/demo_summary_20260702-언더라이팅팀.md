## 도메인 지식 병목을 AI가 어떻게 흡수했는지
- 이전 배치(재보험팀)에서 적발됐던 "abc_costing_principles.md=정의 출처" 오귀속 오류가 이번 배치에서는 재발하지 않음 — driver-recommender 호출 프롬프트에 해당 주의사항을 명시적으로 반영한 결과, result-validator가 별도 재작업 없이 6건 전건을 1회에 통과시켰다. 즉, 검증에서 걸린 오류 패턴이 다음 배치의 재발 방지로 이어지는 것을 확인.

## Phase 0 사전 스캔의 효과 (4개 배치 전체 총괄)
- Phase 0에서 4개 부서 파일을 사전 스캔해 15개 계정을 공통(7)/특정(8)으로 미리 구분해둔 덕분에, 배치를 진행할수록 공통 계정(임차료·소모품비·통신비·여비교통비·교육훈련비)의 재작업량이 줄고 드라이버가 부서 간 일관되게 유지됐다.
- 4개 배치 전체 결과: 총 32건(중복 포함) 중 추가판단 필요 계정은 3종(수선비·전산운영비·보상금) — 이 중 수선비·전산운영비는 4개 부서 전체에서 공통으로 추가판단이 필요하여 원본 데이터에 비고(수선 대상, 인프라 구분 등)를 보강해야 해소 가능하다는 점이 명확히 드러났다.
