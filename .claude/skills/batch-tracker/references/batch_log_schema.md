# batch_log.json 스키마

배치가 언제, 어떤 유형으로 처리되었는지 이력을 남긴다. `track_batch.py complete`가 append 처리 시 기존 이력을 덮어쓰지 않고 누적한다.

**대분류 도입(§0) 이후 구조**: 배치 단위는 계정코드가 아니라 대분류다 — `배치명`은 부서명이 아니라 이 배치에 포함된 대분류 그룹을 가리키고, 건수 집계도 대분류 기준이다.

```json
{
  "batches": [
    {
      "batch_id": "20260711-01",
      "배치명": "공통 대분류 1차 배치",
      "원본파일": "output/batch_20260711-01_validated.json",
      "처리유형": "신규",
      "시작시각": "2026-07-11T10:05:00+09:00",
      "완료시각": "2026-07-11T10:42:00+09:00",
      "총대분류수": 24,
      "추가판단필요대분류수": 3,
      "상태": "완료"
    },
    {
      "batch_id": "20260712-02",
      "배치명": "재보험팀 특정 대분류 피드백",
      "원본파일": "output/batch_20260712-02_validated.json",
      "처리유형": "피드백재반영",
      "시작시각": "2026-07-12T09:00:00+09:00",
      "완료시각": null,
      "총대분류수": 4,
      "추가판단필요대분류수": 0,
      "상태": "진행중"
    }
  ]
}
```

## 필드 설명
| 필드 | 설명 |
|---|---|
| `처리유형` | `신규` \| `갱신` \| `피드백재반영` |
| `상태` | `진행중` \| `완료` \| `실패` |
| `추가판단필요대분류수` | 이 배치에서 "추가판단필요(검토대기)"로 표시된 대분류 수 |

`batch_id` 규칙: `{YYYYMMDD}-{순번}` (피드백 재반영은 원본 batch_id에 이어 새 순번을 부여).
